>
>
>⚠ <span style='color:red'>[`gitlab`启动较慢，请耐心等待，访问地址`http://192.168.8.144:9999`]</span>
>
>⚠ <span style='color:red'>[ 账号`root`，首次密码保留`24`小时，存储在`/usr/local/gitlab-ce18.3.5/etc/initial_root_password`]</span>
>
><br/>

```yaml
services:    
   gitlab:
     image: gitlab/gitlab-ce:18.3.5-ce.0
     container_name: gitlab
     environment:
        GITLAB_OMNIBUS_CONFIG: |
           gitlab_rails['time_zone'] = 'Asia/Shanghai'
        
     ports: 
        - "9998:22"  #SSH
        - "9999:80"  #WEB
        - "8443:443" #Https
      
     volumes:
        - ./etc/:/etc/gitlab          # chmod 777 /usr/local/gitlab-ce18.3.5/etc
        - ./logs/:/var/log/gitlab     # chmod 777 /usr/local/gitlab-ce18.3.5/logs
        - gitlab_data:/var/opt/gitlab # 数据文件
      
     restart: always
    
volumes:
   gitlab_data: ~


```

## ✴ [ 设置 ] 项目群组 :id=H2_2

>[!WARNING|style: flat|label: 简要说明 ]
>
>- 在`GitLab`中 <span style='color:Blue'>[ 群组`Group`]</span> 指的是项目群组 ( 不是用户群组 )
>
>  群组是用来组织和管理多个项目的容器 ( 你可以把群组理解为一个`文件夹`，里面可以包含多个项目，也可以嵌套子群组 )
>
>  <span style='color:red'>[ 群组可以统一管理权限、`CI/CD`配置、成员等 ] [ 群组由`root`统一建立和维护 ]</span>
>
